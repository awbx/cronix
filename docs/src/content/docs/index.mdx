## Five seconds to a scheduled job

```ts
import { createCron, MANIFEST_PATH, TRIGGER_PATH_PREFIX } from "@awbx/cronix-sdk";
import { Hono } from "hono";

const cron = createCron({ app: "billing", baseUrl: "https://billing.example.com", secret: process.env.CRON_SECRET! });

cron.register({
  name: "reconcile-payments",
  schedule: "*/15 * * * *",
  auth: { secret_refs: ["env:CRON_SECRET"] },
  handler: async (ctx) => { /* your work */ return { ok: true }; },
});

const app = new Hono();
app.all(MANIFEST_PATH, (c) => cron.handle(c.req.raw));
app.all(`${TRIGGER_PATH_PREFIX}:name`, (c) => cron.handle(c.req.raw));
```

Then from your laptop or CI:

```bash
cronix apply --manifest https://billing.example.com/.well-known/cron-manifest \
  --backend crontab --crontab-path /etc/crontab \
  --trigger-bin /usr/local/bin/cronix --secret-ref env:CRON_SECRET
```

That's it. Your `*/15 * * * *` line lives in your code; `cron(8)` actually fires it; `cronix trigger` signs the request and POSTs back to your handler with retries, locks, and HMAC.

## Why it's different

<CardGrid>
  <Card title="Schedule lives with the handler" icon="pencil">
    The cron expression is a property of the function it triggers, not a row in some other system. Reviewers see schedule changes in the same diff as code changes.
  </Card>
  <Card title="Your app is the source of truth" icon="approve-check">
    cronix reads your manifest at apply time and reconciles the host scheduler to match. Drift between what's declared and what's installed is detectable with one command.
  </Card>
  <Card title="Native scheduler does the firing" icon="setting">
    No background process, no broker. cron, systemd-timer, kubelet, EventBridge — whatever the host already runs — does the work. cronix only manages entries.
  </Card>
  <Card title="HMAC-signed every fire" icon="seti:lock">
    Stripe-style signatures (timestamp + sha256) on every request. Constant-time verify. Multiple secrets supported for zero-downtime rotation.
  </Card>
  <Card title="Multi-backend" icon="puzzle">
    crontab, systemd-timer, Kubernetes CronJob, AWS EventBridge Scheduler — same manifest, same signing contract, swap with one flag.
  </Card>
  <Card title="Polyglot from day one" icon="seti:typescript">
    TypeScript SDK + Go SDK conformance-tested against shared spec vectors. Python, Ruby, and others welcome — the protocol is the product.
  </Card>
</CardGrid>

## How it works

```
              app (your service)
              GET /.well-known/cron-manifest
              POST /api/v1/scheduled/<name>
                       │
                       │ (1) read manifest
                       ▼
                cronix apply (Go)
                       │
                       │ (2) install/update/delete entries
                       ▼
              host scheduler (crontab / systemd / k8s CronJob / EventBridge)
                       │
                       │ (3) invoke on schedule
                       ▼
                cronix trigger (Go)
                       │   • acquires lock
                       │   • signs HMAC
                       │   • POSTs to your handler
                       │   • applies timeout + retries
                       ▼
                app handler (verifies signature, dedupes by run-id)
```

The protocol is the product. The reconciler and SDKs are reference implementations — anyone can ship a conforming SDK by passing the shared `manifest-vectors.json` and `auth-vectors.json`.
